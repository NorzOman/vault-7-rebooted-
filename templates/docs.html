Vault - 7 [2.0] API Documentation

Authentication

GET /get_token

Get an authentication token that is valid for 1 hour.

curl -X GET /get_token

Health Check

GET /check_health

Check if the API is running properly.

curl -X GET /check_health

File Scan

POST /file_scan

Scan a file to check if it contains malicious content.

curl -X POST /file_scan \
    -H "Content-Type: application/json" \
    -d '{"token": "your-token", "file": "file_to_scan"}'

Message Scan

POST /message_scan

Scan a message to check if it contains malicious content.

curl -X POST /message_scan \
    -H "Content-Type: application/json" \
    -d '{"token": "your-token", "message": "Message to scan"}'

URL Scan

POST /url_scan

Perform a comprehensive security scan of a URL.

curl -X POST /url_scan \
    -H "Content-Type: application/json" \
    -d '{"token": "your-token", "url": "https://example.com/path"}'

The URL scan provides:

  • Domain information (registrar, creation date, etc.)
  • Security issues found
  • Risk rating (0-10)
  • Risk level (SAFE, CAUTION, or DANGEROUS)

IP Scan

POST /ip_scan

Scan an IP address to check if it's safe or malicious.

curl -X POST /ip_scan \
    -H "Content-Type: application/json" \
    -d '{"token": "your-token", "ip_address": "192.168.1.1"}'

Important Notes

  • All scanning endpoints require a valid token from /get_token
  • Tokens expire after 1 hour
  • All responses are in JSON format
